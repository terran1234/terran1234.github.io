자원 공유

NFS로 네트워크 폴더를 공유하고, Samba로 리눅스↔윈도우 간 파일을 공유하는 설정 정리

1. NFS (Network File System)

네트워크로 다른 컴퓨터의 폴더를 내 컴퓨터처럼 쓰는 것.

dnf install -y nfs nfs-utils   // 서버, 클라이언트 양쪽에 설치

서버 설정

systemctl start nfs-server
systemctl start rpcbind

mkdir /nfsserver
chmod 756 /nfsserver

vi /etc/exports
// /nfsserver 클라이언트IP

exportfs -a
exportfs

firewall-cmd --permanent --add-port=2049/tcp
firewall-cmd --reload

클라이언트에서 폴더를 생성하면 서버에도 자동으로 반영됨(동기화).

클라이언트 설정

systemctl start rpcbind

mkdir /nfsclient
mount -t nfs [Server IP]:/nfsserver /nfsclient/

df
df -h   // 용량 단위로 확인

umount /nfsclient   // 해제 전 rpcbind 꺼야 함

2. Samba

리눅스↔윈도우 파일 공유 시스템. nmbd(137,138/udp, 139/tcp), smbd(445/tcp — 대부분 처리).

dnf install -y samba samba-common samba-client

mkdir /sambatest
chmod 777 /sambatest

useradd sambauser
smbpasswd -a sambauser   // Samba 사용자 등록+비밀번호
passwd sambauser          // 시스템 로그인 비밀번호
vi /etc/samba/smb.conf
[Sambashare]
    path = /sambatest
    browsable = yes
    writable = yes
    guest ok = yes
    create mask = 0777
    directory mask = 0777
testparm      // 설정 오류 검사
smbstatus     // 상태 확인
systemctl start smb nmb

firewall-cmd --permanent --add-service=samba
firewall-cmd --reload

윈도우 실행창에서 \\[삼바 서버 IP]로 접속해 폴더 확인. SELinux가 기본적으로 막고 있는 경우가 있음.

sestatus

vi /etc/selinux/config
SELINUX=disabled   // 재부팅 필요

테스트 후에는 다시 enforcing(강제 적용 중)으로 되돌려야 함.
