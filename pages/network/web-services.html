HTTP, HTTPS, FTP, DNS

아파치·SSL 인증서·vsftpd·BIND로 4가지 핵심 서버를 직접 구축한 기록

1. HTTP 서버

웹에서 데이터를 주고받는 규칙, 80번 포트/tcp 사용.

dnf install -y httpd mariadb-server php
systemctl start httpd
firewall-cmd --permanent --add-port=80/tcp
firewall-cmd --reload
<html><body>Hello world!</body></html>
// /var/www/html/test.html 경로에 생성

2. HTTPS 서버

HTTP에 SSL/TLS 암호화를 적용한 보안 통신 프로토콜. 요즘은 TLS를 쓰지만 관습적으로 SSL이라 부름.

mkdir /etc/httpd/conf/ssl
cd /etc/httpd/conf/ssl
openssl req -newkey rsa:2048 -nodes -keyout /etc/httpd/conf/ssl/test.com.key -x509 -days 365 -out /etc/httpd/conf/ssl/test.com.crt
openssl x509 -noout -text -in test.com.crt

rsa:2048=비대칭키 2048비트, -nodes=비밀번호 없이 키 생성, -x509=인증서 생성, -days=유효기간.

dnf install -y mod_ssl
httpd -M

vi /etc/httpd/conf.d/ssl.conf
SSLCertificateFile /etc/httpd/conf/ssl/test.com.crt
SSLCertificateKeyFile /etc/httpd/conf/ssl/test.com.key

systemctl restart httpd
firewall-cmd --permanent --add-service=https
firewall-cmd --reload

공인 기관(CA)이 아닌 직접 만든 인증서라 브라우저에서 경고 문구가 뜸.

3. FTP 서버

파일 업로드/다운로드용 프로토콜. 기본 포트 21(제어), 내부적으로 20(전송) 동작.

  • Active Mode — 서버가 클라이언트로 데이터 연결 시도 (방화벽에 막히기 쉬움)
  • Passive Mode — 클라이언트가 서버로 연결 시도, 1024 이상(보통 50000~55000) 포트 사용, 방화벽 환경에서 더 많이 사용
dnf install -y vsftpd
systemctl start vsftpd
systemctl enable vsftpd

firewall-cmd --permanent --add-service=ftp
firewall-cmd --reload
useradd ftptest
passwd ftptest
mkdir /home/ftptest
touch /home/ftptest/ftp.txt

FileZilla 클라이언트로 접속해 확인. /home 디렉토리에 계정 생성됨.

FTPS 서버 (SSL 암호화)

openssl req -newkey rsa:2048 -nodes -keyout /etc/vsftpd/vsftpd.pem -x509 -days 365 -out /etc/vsftpd/vsftpd.pem
vi /etc/vsftpd/vsftpd.conf
ssl_enable=YES
rsa_cert_file=/etc/vsftpd/vsftpd.pem
force_local_logins_ssl=YES
force_local_data_ssl=YES
ssl_tlsv1=YES
allow_anon_ssl=NO
pasv_enable=YES
pasv_min_port=50000
pasv_max_port=55000

systemctl restart vsftpd
firewall-cmd --permanent --add-port=50000-55000/tcp
firewall-cmd --reload

vsftpd는 SSL 기능이 내장돼 있어 HTTPS처럼 별도 모듈 설치가 필요 없음.

SFTP

FileZilla에서 SFTP로 설정, 포트 22번 사용. 리눅스는 SSH가 기본 설치돼 있어 추가 설정 없이 바로 사용 가능.

4. DNS

도메인 이름을 IP 주소로 바꿔주는 시스템.

레코드의미
A도메인 → IPv4
AAAA도메인 → IPv6
CNAME별명(다른 도메인으로 연결)
MX메일 서버
NSDNS 서버
SOA전체 설정 정보
dnf install -y bind bind-utils
vi /etc/named.conf
listen-on port 53 { any; };
allow-query     { any; };
recursion no;
zone "test.com" IN {
    type master;
    file "test.com.zone";
    allow-update { none; };
};
vi /var/named/[도메인].com.zone
$TTL 86400
@   IN  SOA  ns.test.com. root.test.com. ( 20260402 24H 15M 48H 86400 );
    IN  NS   ns.test.com.
@   IN  A    [DNS IP]
ns  IN  A    [NS 서버 IP]
www IN  A    [WEB 서버 IP]
named-checkconf /etc/named.conf
named-checkzone [도메인].com /var/named/[도메인].com.zone

firewall-cmd --permanent --add-service=dns
firewall-cmd --reload
systemctl start named
systemctl enable named

테스트 시 클라이언트의 로컬 DNS 설정도 직접 구축한 NS 주소로 수동 변경 필요.
