1. 로그 기본 개념

  • 시스템에서 일어난 일을 기록해 놓은 파일, 기본 위치 /var/log/
  • logrotate — 로그를 주기적으로 교체/정리하는 기능, 설정 파일은 /etc/logrotate.d
  • 로그 서버 = 관리 편의 + 증거 보존 목적

Facility (발생 위치)

Facility설명
auth로그인, 인증 관련
authpriv인증 중 민감 정보
cron예약 작업 실행 로그
daemon백그라운드 서비스 로그
kern커널 로그
mail이메일 관련 로그
user일반 프로그램 로그
local0~7직접 정의해서 쓰는 로그

Priority (긴급도)

Priority설명

중요 로그 파일

로그이유확인 명령어
messages시스템 전체 상황(기본 로그)cat messages

lastlog, wtmp, btmp는 바이너리 형식이라 cat으로 열면 깨져서 출력됨.

2. rsyslog 서버 구축

클라이언트에서 발생한 로그를 서버로 전송하고, 서버에서 실시간으로 확인하는 과정.

dnf install -y rsyslog rsyslog-doc   // 서버, 클라이언트 양쪽
vi /etc/rsyslog.conf
$ModLoad imudp
$InputUDPServerRun 514
$ModLoad imtcp
$InputTCPServerRun 514

firewall-cmd --permanent --add-port=514/tcp
firewall-cmd --permanent --add-port=514/udp
firewall-cmd --reload
systemctl start rsyslog
systemctl enable rsyslog
dnf install -y lsof net-tools
lsof -i tcp:514
lsof -i udp:514
netstat -npl | grep 514
// 클라이언트에서 전송 대상 설정
vi /etc/rsyslog.conf
*.* @@[서버 IP]:514   // TCP
*.* @[서버 IP]:514    // UDP

systemctl restart rsyslog
// 확인
tail -f /var/log/messages
systemctl restart sshd   // 클라이언트에서 이벤트 발생시켜 서버 로그 갱신 확인

3. DB 연동

dnf install -y mariadb-server
systemctl start mariadb
systemctl enable mariadb

dnf install -y rsyslog-mysql
mysql -u root -p < /usr/share/doc/rsyslog/mysql-createDB.sql

grant all on Syslog.* to '[사용자명]'@'%' identified by 'password';
flush privileges;
vi /etc/rsyslog.conf
$ModLoad ommysql
*.*:ommysql:127.0.0.1,Syslog,rsyslog,password

systemctl restart rsyslog
firewall-cmd --permanent --add-port=3306/tcp
firewall-cmd --reload
// 확인
mysql -u root -p
use Syslog;
select * from SystemEvents;

4. LogAnalyzer (웹 시각화)

dnf install -y httpd php php-mysqli wget tar

wget -P /tmp http://download.adiscon.com/loganalyzer/loganalyzer-4.1.13.tar.gz
tar -xzvf /tmp/loganalyzer-4.1.13.tar.gz -C /tmp
mkdir /var/www/html/loganalyzer
cp -r /tmp/loganalyzer-4.1.13/src/* /var/www/html/loganalyzer
cp /tmp/loganalyzer-4.1.13/contrib/configure.sh /var/www/html/loganalyzer
bash configure.sh
touch config.php
chmod 666 config.php

systemctl restart httpd

웹으로 http://[서버 IP]/loganalyzer/ 접속 후 단계별 설정 진행. SELinux 정책 때문에 다음 명령어들이 추가로 필요할 수 있음.

chcon -h -t httpd_sys_script_rw_t config.php

setsebool -P httpd_can_network_connect_db 1
systemctl restart httpd

Source Type을 MySQL Native로 설정하고 DB 정보 입력 후, 클라이언트에서 이벤트 발생시켜 웹페이지에 실시간 갱신되는지 확인.

5. GREP

파일이나 출력에서 원하는 문자열을 찾아주는 명령어.

grep "찾을내용" 파일명
grep "error" /var/log/messages
