라우팅 설정

정적/동적 라우팅 개념, AD·Metric 비교, RIP·EIGRP 프로토콜 설정 정리

1. 라우팅 프로토콜 기본 개념

서로 다른 네트워크로 데이터를 보내기 위해 최적의 경로를 찾고 전달하는 것.

Static Routing (정적 라우팅)

  • 관리자가 직접 경로를 설정
  • 단순하지만 규모가 커지면 관리 어려움

Dynamic Routing (동적 라우팅)

  • 라우터끼리 자동으로 경로를 학습
  • 대표 프로토콜: RIP, EIGRP, OSPF, BGP

2. 정적 라우팅 설정

ip route [목적지 네트워크] [서브넷 마스크] [다음 홉 IP]

예: 192.168.200.0 네트워크로 가려면 10.0.0.2 라우터를 거치라는 뜻. 양쪽 라우터에 모두 설정해야 통신 가능.

설정 확인 / 삭제

show running-config
show ip route      // S(static)는 정적 라우팅 설정을 의미

no ip route [목적지 네트워크] [서브넷 마스크] [다음 홉 IP]

3. AD / Metric

show ip route 출력의 대괄호 안 숫자 중 왼쪽은 AD(신뢰도), 오른쪽은 Metric(비용). 서로 다른 라우팅 프로토콜끼리 경쟁할 때는 AD가 더 낮은 쪽이 살아남고, 같은 프로토콜 안에서는 Metric을 비교해 가장 좋은 경로를 선택.

🖼 스크린샷 자리 — AD 값 설정 및 tracert 결과
  • 라우팅 추가 명령어 뒤에 AD 값만 붙이면 설정 가능
  • AD 값이 같으면 라우팅 테이블에 같이 올라감(부하 분산)
  • 더 낮은 AD 값의 정보만 라우팅 테이블에 반영되고, 나머지는 show running-config로만 확인 가능
  • tracert [주소]로 실제 경로 확인 — AD 값을 바꾸면 선택되는 경로도 바뀌는 것을 실습으로 확인

4. 동적 라우팅 (RIP) 설정

  • 홉 수가 최적 경로(메트릭 값)를 결정하는 프로토콜 — 홉이 적어도 저속 회선이면 비효율적일 수 있음
  • 15홉 제한, 16부터는 unknown 처리
  • 30초마다 hello packet을 주고받아 업데이트
router rip
network 192.168.10.0
network 10.0.0.0
version 2
no auto-summary

router rip로 프로토콜 모드 진입, network로 참여시킬 네트워크 등록. RIP는 서브넷/와일드 마스크가 필요 없고, 라우팅 이웃은 양방향 동의로 형성됨. 기본값은 버전 1이지만 서브넷 정보를 못 넘겨서(classful) version 2로 설정. no auto-summary까지 해줘야 클래스 단위 자동 축약이 일어나지 않고 서브넷 그대로 광고됨.

no router rip   // RIP 전체 설정 삭제

5. 동적 라우팅 (EIGRP) 설정

대역폭과 지연시간을 기준으로 최적 경로를 선택하는 Cisco 전용 프로토콜로, RIP보다 훨씬 효율적.

router eigrp [AS 번호]
network [네트워크 주소] [Wildcard Mask]
no auto-summary

AS 번호는 독립된 네트워크 묶음의 이름표. Wildcard Mask는 서브넷 마스크를 반전시킨 것으로 EIGRP는 이걸 사용. 패킷 트레이서 최신 버전의 EIGRP는 이미 no auto-summary 상태라 입력해도 반응 없을 수 있음.

no router eigrp [as 번호]   // 삭제

재분배 (Redistribute)

router eigrp 20
redistribute eigrp 10

router eigrp 10
redistribute eigrp 20

서로 다른 라우팅 프로토콜(예: eigrp 10 ↔ eigrp 20)끼리 경로를 공유해서 통신. 하나의 경계 라우터에 위 명령어를 다 입력하고 network도 추가해야 함. 양방향 재분배가 원칙. D EX = 재분배로 들어온 외부 경로 (신뢰도 낮음, AD 170).
