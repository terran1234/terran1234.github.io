ACL, NAT

접근 제어 목록으로 트래픽을 필터링하고, 사설·공인 IP 주소를 변환하는 설정 정리

1. ACL (Access Control List)

정의 및 동작 방식

  • 특정 트래픽을 허용(permit)하거나 차단(deny)하는 필터
  • 위에서 아래로 조건을 하나씩 검사
  • deny any가 생략되어 있어서 지정하지 않은 패킷은 자동 차단
  • 주로 L3 장비(라우터, L3 스위치)에 설정

Inbound / Outbound

  • Inbound — 패킷이 들어오면 ACL을 먼저 검사한 뒤, 허용된 패킷만 라우팅 테이블을 참고해 목적지로 전달
  • Outbound — 라우팅 테이블로 출구 인터페이스가 결정된 이후, 나가기 전에 ACL 정책 검사

종류

기능방식예시
StandardNumberedaccess-list 10
StandardNamedip access-list standard
ExtendedNumberedaccess-list 100
ExtendedNamedip access-list extended

표준(Standard, 1~99)은 출발지 IP만 확인. 확장(Extended, 100~199)은 출발지+목적지+프로토콜+포트까지 확인. 번호 기반은 수정이 거의 불가능해 전체 삭제 후 재작성해야 하고, 이름 기반은 전용 모드로 들어가 수정 가능.

Standard & Numbered 설정

access-list 10 permit host 192.168.0.10
access-list 10 permit any

interface fa0/0
ip access-group 10 out

access-list로 누굴 허용/차단할지 정하고, ip access-group으로 어디에 적용할지 정함. 목적지와 가장 가까운 인터페이스에 적용하는 게 불필요한 점검을 줄여 효율적이며, 주로 outbound 사용.

Extended & Numbered 설정

access-list 100 permit tcp 192.168.0.10 0.0.0.0 192.168.1.100 0.0.0.0 eq 80
access-list 100 permit ip any any
명령어의미예시
eq같다eq 80
gt초과gt 1023
lt미만lt 1024
neq같지 않음neq 23
range범위range 20 30

확인 및 삭제

show access-lists

no access-list 10                          // Standard Numbered 삭제
no access-list 100                         // Extended Numbered 삭제
no ip access-list standard [이름]           // Standard Named 삭제
no ip access-list extended [이름]           // Extended Named 삭제

2. NAT (Network Address Translation)

  • 내부 사설 IP를 공인 IP로 변환
  • static / dynamic 방식 존재, 기본은 1:1 매핑
  • /24 vs /24는 매핑 가능하지만, /26 vs /24는 1:1 매핑 불가능

종류

  • Dynamic NAT — ACL 설정 필요, 공인 IP Pool 사용, 동적 1:1 매핑
  • Static NAT — 특정 IP를 고정 매핑
  • PAT (Port Address Translation) — n:m 구조, 하나의 공인 IP를 여러 내부 IP가 포트로 구분해 공유

Dynamic NAT 설정

// 1. ACL 설정
access-list 10 permit 192.168.1.0 0.0.0.255

// 2. NAT Pool 생성
ip nat pool nattest 5.5.5.1 20.20.20.254 netmask 255.255.255.0

// 3. ACL + Pool 연결
ip nat inside source list 10 pool nattest

// 4. 인터페이스 설정
ip nat inside   (/fa)
ip nat outside  (/se)

누굴 바꿀지(access-list) → 뭘로 바꿀지(ip nat pool) → 둘을 연결(ip nat inside source) → 내부/외부 인터페이스 지정(ip nat inside/outside) 순서.

Static NAT 설정

ip nat inside source static 192.168.1.100 20.20.20.1

내부(사설) IP 192.168.1.100을 외부(공인) IP 20.20.20.1로 1:1 고정 매핑.
