2. 커널 설정

sysctl
sysctl -w vm.swappiness=40

vi /etc/sysctl.conf   // 영구 반영

sysctl로 바꾼 값은 일시적이라 재부팅하면 사라짐. 영구 반영하려면 설정 파일 편집 필요.

free -h
cat /proc/meminfo
cat /proc/version
cat /proc/cpuinfo

메모리(RAM), 커널 버전, CPU 정보 확인.

3. DB 보안 (사용자·권한)

create user 'estuser'@'localhost' identified by '1234';
create database est default character set utf8;
drop user hoon;
drop user 'winuser'@'172.30.1.76';
grant all privileges on est_db.* to 'estuser'@'localhost';
grant create, drop on est_db.* to 'winuser'@'172.30.1.76';
flush privileges;
show grants;
show grants for 'estuser'@'localhost';

all privileges가 아니면 privileges는 빼야 함. show grants는 접속 계정 본인 권한만 출력. grant로 사용자 생성과 권한 부여를 동시에 처리 가능.

revoke all privileges on *.* from 'edushare'@'localhost';

4. DB 복제 (Master-Slave)

Master 서버 설정

vi /etc/my.cnf.d/mariadb-server.cnf
server-id=1
log_bin=/var/log/mariadb/mysql-bin.log
expire_logs_days=10
max_binlog_size=10M

touch /var/log/mariadb/mysql-bin.log
systemctl restart mariadb

log_bin은 DB 작업(INSERT, UPDATE 등)을 기록하는 설정 — 이걸 켜면 마스터 역할.

Slave 권한 부여 & 서버 설정

grant replication slave on *.* to 'slave_db'@'%' identified by '1234';
flush privileges;
systemctl restart mariadb
vi /etc/my.cnf.d/mariadb-server.cnf
server-id=2

systemctl restart mariadb

Slave DB 설정

show variables like 'server_id';

change master to master_host="172.30.1.24",
  master_user="slave_db",
  master_password="1234",
  master_port=3306,
  master_log_file="mysql-bin.000004",
  master_log_pos=342;

show slave status\G

CHANGE MASTER 명령을 실행하면 슬레이브 역할로 동작.
