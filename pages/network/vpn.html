VPN, 터널링, IPSec

GRE 터널링과 IPSec/ISAKMP 기반 site-to-site VPN 구성 정리

1. VPN (Virtual Private Network)

인터넷 같은 공용망 위에서 안전하게 통신하기 위해 암호화된 가상 사설망을 만드는 기술.

  • Site-to-Site VPN — 네트워크↔네트워크 + 암호화. 본사↔지사처럼 두 LAN을 인터넷 위 암호화된 가상 터널로 연결
  • GRE (Generic Routing Encapsulation) — 서로 다른 네트워크 간 데이터를 캡슐화해 터널 형태로 전달하는 프로토콜. 암호화는 안 하고 터널(길)만 만들어줌
  • IPSec — IP 계층에서 데이터를 암호화·인증해 안전하게 통신하는 보안 프로토콜. 멀티캐스트가 안 되서 동적 라우팅에 못 쓰므로 GRE로 감싸서 전송

2. 터널링 설정 (Site-to-Site, GRE)

interface tunnel 100
ip address 10.10.10.10 255.255.255.0

tunnel mode gre ip

tunnel source [출발지 인터페이스]
tunnel destination [목적지 물리 IP 주소]

ip route 172.16.0.0 255.255.255.0 10.10.10.20

가상 터널 인터페이스를 만들고 IP를 부여(터널 번호는 양쪽이 같을 필요 없음) → 캡슐화 방식(GRE) 지정 → 터널 시작/끝 위치 지정 → 터널을 통해 갈 목적지 경로 추가. 반대편 라우터에도 동일하게 설정.

🖼 스크린샷 자리 — 터널링 설정 후 통신 확인

터널링 설정 후에는 물리 회선이 아닌 가상 회선을 통해 통신하는 걸 확인할 수 있음. 실제로는 물리 회선을 타지만 중간 경로를 은닉하는 효과(보안성 자체를 높이는 건 아님).

show ip interface tunnel 100

3. VPN Site-to-Site 설정 (IPSec)

큰 흐름: 암호화할 트래픽 지정(access-list) → ISAKMP 정책 생성(인증/암호화/해시 방식) → 상대와 쓸 키·대상 지정(crypto isakmp key) → 데이터 암호화 방식 지정(crypto transform-set) → 위 설정들을 하나로 묶기(crypto map) → 실제 인터페이스에 적용.

// 본사 라우터 (괄호 안은 지사 라우터 값)
access-list 100 permit gre host 1.1.1.1 host 2.2.2.1
access-list 100 permit gre 192.168.100.0 0.0.0.255 host 2.2.2.1

crypto isakmp policy 10
authentication pre-share
encryption 3des
hash md5
crypto isakmp key 1234 address 2.2.2.1
crypto ipsec transform-set test esp-3des esp-md5-hmac

crypto map vpn-map 10 ipsec-isakmp
match address 100
set peer 2.2.2.1
set transform-set test

interface s0/3/0
crypto map vpn-map

세밀한 제어를 위해 access-list로 암호화할 트래픽을 지정. IPSec(암호화)는 실제 공인 IP(물리 인터페이스) 기준으로 동작. ISAKMP는 준비 단계, IPSec은 실제 데이터 암호화 단계. authentication pre-share는 미리 정한 비밀번호로 상호 인증하는 방식이며, crypto isakmp key로 누구와 키를 쓸지 지정. esp는 실제 데이터 보호 방식(IPSec 프로토콜). 반대편 라우터에도 동일하게 설정 필요.

show crypto ipsec sa

encaps / decaps 숫자가 올라가는지로 정상 동작 여부 확인.
