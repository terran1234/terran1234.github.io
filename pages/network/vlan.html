VLAN 설정

OSI 7계층, 패킷 트레이서 기초 명령어, 서브넷 계산, VLAN·Trunk·VTP·서브 인터페이스 구성 정리

1. OSI 7계층 순서

물리 → 데이터 링크 → 네트워크 → 전송 → 세션 → 표현 → 응용

아래(물리)에서 위(응용)로 갈수록 사용자에 가까움.

2. 패킷 트레이서 기초 명령어

암호화 설정 및 취소

enable password [비밀번호]   // 약한 암호화
enable secret [비밀번호]     // 강한 암호화, enable password보다 우선 적용

no service password-encryption   // 암호화 취소, 기존 암호는 안 풀리고 새로 설정하는 것만 평문 저장

설정 확인

🖼 스크린샷 자리 — show version / show startup-config / show running-config
show version           // 하드웨어 정보 확인 (예: 라우터 2811)
show startup-config    // 저장된 설정 확인 (없으면 not present)
show running-config    // 현재 적용된(임시) 설정 확인

설정 저장

라우터는 전원이 꺼지면 설정이 사라질 수 있어 재설정 문제를 방지하기 위해 저장이 필요함.

copy running-config startup-config
write memory

RAM(임시) → NVRAM(영구 저장)으로 옮기는 과정. 두 명령어는 차이 없음.

3. 서브넷

네트워크를 효율적으로 사용하기 위해 나눈 작은 네트워크.

서브넷 예시 — /26 = 호스트 비트 6개

  • 전체 IP 개수 → 2⁶ = 64
  • 네트워크 주소(첫 번째)와 브로드캐스트(마지막) 주소는 제외
  • 실제 사용 가능 → 64 - 2 = 62

192.168.100.0/24를 4개 서브넷으로 분할

4개 서브넷 필요 → 2비트(2²=4) 추가 사용 → 서브넷 마스크 /26 → 남은 호스트 비트 6비트(2⁶=64, 블록 크기)

192.168.100.0   ~ 192.168.100.63
192.168.100.64  ~ 192.168.100.127
192.168.100.128 ~ 192.168.100.191
192.168.100.192 ~ 192.168.100.255

Q. /26이면 255.255.255.192인데, 마지막 옥텟도 192부터 시작하나요?
A. IP 주소와 서브넷 마스크는 별개 — /25, /26이라고 해서 특정 숫자로 시작하는 건 아님.

4. VLAN

종류

  • Port 기반 VLAN (가장 많이 사용)
  • IP 기반 VLAN
  • MAC 기반 VLAN

show vlan 명령어로 확인하면 기본적으로 모든 포트는 VLAN 1에 할당되어 있음.

Trunk 모드

access 모드에서는 태그를 안 붙여서 하나의 VLAN만 통신 가능. 스위치↔스위치 또는 라우터 연결 회선에서는 다양한 VLAN이 통신 가능해야 해서, 태그를 붙여 여러 VLAN이 통신 가능하도록 트렁크 모드로 설정.

switchport mode trunk

확인은 show vlan이 아니라 show interfaces trunk로 (trunk는 소속이 아니라서 show vlan으로 안 보임).

수동 설정 방법

vlan [번호]
name [이름]

VLAN 번호를 생성하고 이름을 붙임.

switchport mode access
switchport access vlan 10

인터페이스 모드로 들어가서, 단말(PC) 전용 포트로 선언 후 VLAN 10에 할당.

5. VLAN 동기화 프로토콜 (VTP)

새 VLAN을 추가할 때마다 모든 스위치에 수동 설정하는 건 비효율적이므로, VLAN 정보를 자동 동기화하기 위해 VTP(VLAN Trunking Protocol) 사용.

모드설명
ServerVLAN 생성 + 배포
Client받아서 사용
Transparent독립 (공유 안 함)
vtp domain [그룹명]
vtp password [비밀번호]

vtp mode server
vtp mode client
vtp mode transparent

동기화하려면 도메인과 비밀번호가 같아야 함. 스위치끼리 연결 회선을 트렁크 모드로 연결해야 VTP 작동.

show vtp status

6. 서브 인터페이스

하나의 물리 포트를 여러 개처럼 나눠서 쓰는 것. VLAN과 연결할 때 물리 회선을 늘리는 대신 가상 회선을 구성해 자원을 효율적으로 활용.

interface fa0/0.10
encapsulation dot1Q 10
ip address 192.168.10.126 255.255.255.128

인터페이스 진입 명령어 뒤에 .[숫자]를 붙이면 서브 인터페이스 모드로 진입 (가시성을 위해 VLAN 번호와 동일하게 지정). 서브 인터페이스는 부모 인터페이스 상태를 따라가서 no shutdown은 사용하지 않음. 라우터는 VLAN 태그를 보고 해당 서브인터페이스로 분기하며, 이를 위해 encapsulation dot1Q [VLAN 번호]를 설정. 게이트웨이 역할을 위해 서브 인터페이스에 IP 주소도 부여.
