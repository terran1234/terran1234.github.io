리눅스 계정 관리

계정 정보 구조, 비밀번호 암호화, 권한(umask), 파일 검색·삭제, 히스토리 확인 등 계정 관련 명령어 정리

1. 계정 정보 확인

cat /etc/passwd
// [사용자 이름]:x:[UID]:[GID]:root:/root:/bin/bash

x는 예전엔 비밀번호가 있던 자리인데, 지금은 보안 때문에 /etc/shadow에 숨겨져서 표시만 남은 것. /bin/bash는 로그인하면 실행될 쉘.

cat /etc/shadow
// [이름]:$6$솔트$해시값::0:99999:7:::

$6$=SHA-512 해시 방식($1$은 MD5). 솔트 값과 해시 값(비밀번호+솔트를 SHA-512로 계산한 결과)이 이어짐. 이후 숫자는 순서대로 최소 변경 기간(0), 만료 기간(99999), 경고 기간(7, 만료 7일 전부터 매일 경고).

비밀번호 변경 / 계정 전환

passwd    // root 접속 상태면 아이디 생략, root 비밀번호 변경
su [계정명]   // 계정 전환
hostnamectl status
hostnamectl set-hostname Server   // 재부팅 또는 su로 계정 전환 필요

소유자 변경

chown root ttt.txt
chgrp root ttt.txt
chown root:root ttt.txt   // 소유자+그룹 동시 변경

2. 권한 관련

cd /tmp
umask
umask [4자리 숫자]
touch test.txt
mkdir test
chmod 777 test.tx

umask는 새로 만들 때 빼버릴 권한(기본 0022). 파일 기본값(666), 디렉토리 기본값(777)에서 umask를 뺀 값이 초기 권한 — 그래서 파일은 644, 디렉토리는 755가 됨. chmod는 생성 후 권한을 수정할 때 사용.

3. 파일 관련

ls -al /etc
-rw-r--r--  ...
drwxr-xr-x  ...

맨 앞 글자로 종류 구별: -=일반 파일, d=디렉토리, l=링크.

ls -al /etc
ls -ltr /etc
ls -lhS /etc
ls -ld /etc

-l=길게, -a=숨김파일 포함, -t=최신순, -r=역순, -h=용량 사람이 보기 쉽게, -S=큰 파일 먼저, -d=디렉토리 자체 권한 확인.

find / -name sshd_config
rmdir test3/test4/test5      // 비어있는 디렉토리만 삭제
rm -rf test3/test/4          // -r 재귀 삭제, -f 확인 없이 강제 삭제

4. 도움 / 전원 / 기록 / 출력 / 내 정보

man ls   // 매뉴얼 보기
shutdown -h now   // halt, 즉시 종료
shutdown -r now   // reboot, 즉시 재시작
history
cat /root/.bash_history

history -w   // 현재 기록을 파일에 저장
history -c   // 현재 히스토리 삭제 (파일 내용은 유지)
echo "test"   // 터미널에 글자 출력
date       // 날짜/시간
id         // UID, GID, 소속 그룹
who        // 현재 로그인한 사용자 목록
whoami     // 지금 내가 누구인지
pwd        // 현재 경로
