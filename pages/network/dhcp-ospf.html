DHCP, OSPF

IP 자동 할당 프로토콜과 최단 경로 기반 동적 라우팅 프로토콜 정리

1. DHCP

정의

  • 자동으로 IP 주소를 할당해주는 프로토콜
  • 네트워크에 필요한 기본 정보를 전부 자동 설정

동작 흐름

  • PC가 IP 요청 (DHCP Discover)
  • DHCP 서버가 제안 (Offer)
  • PC가 선택 (Request)
  • 서버가 확정 (ACK)

설정 방법

service dhcp

ip dhcp excluded-address 192.168.100.101 192.168.100.110

ip dhcp pool test
 network 192.168.100.0 255.255.255.0
 default-router 192.168.100.254
 dns-server 8.8.8.8

라우터를 DHCP 서버로 사용하겠다고 선언 후, excluded-address로 제외할 IP 범위를 지정. ip dhcp pool로 나눠줄 IP 대역의 이름을 지정하고 설정모드 진입, network로 PC들이 받을 IP 범위, default-router로 라우터 자신의 IP, dns-server로 DNS 서버를 설정.

설정 확인

show ip dhcp binding    // 어떤 MAC 주소에 어떤 IP가 할당됐는지 확인
show ip dhcp conflict   // IP 충돌 발생 시 확인

2. OSPF (Open Shortest Path First)

정의

  • 최단 경로를 계산해서 라우팅하는 프로토콜
  • RIP보다 빠르며, 이름처럼 속도보다는 정확하게 최단 경로를 계산하는 게 핵심

핵심 구조

  • 네트워크를 구역(Area)으로 나눔
  • Area 0 (백본 영역) — 필수
  • Area 1, 2, 3 — 선택
  • 모든 Area는 반드시 Area 0과 연결되어야 함

설정 방법

router ospf 10

network 10.10.10.0 0.0.0.255 area 0

router ospf [프로세스 ID]는 각 라우터마다 값이 달라도 되며(AS 번호와 다름), 설정 모드로 진입. network [네트워크 주소] [와일드 마스크] area [번호]로 광고할 네트워크 대역을 설정. 별도로 no auto-summary는 필요 없음.
