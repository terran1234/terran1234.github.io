윈도우 서버 2022, 패스워드 정책

Windows Server IIS로 FTP·DNS 구축, 리눅스 비밀번호 복잡도 정책 설정 정리

1. 윈도우 서버 2022

FTP 구축

  • FTP 설치 — Server Manager의 Add roles and features에서 FTP 설치
  • 사이트 추가 — Server Manager → IIS 도구 → Sites 폴더 우클릭 → Add FTP Site
  • C:\inetpub\ftproot — IIS FTP 기본 루트 폴더

Authentication(인증) — Anonymous: 아이디 없이 접속 / Basic: 아이디·비밀번호 로그인

Authorization(인가) — All Users / Specified users / Anonymous users

Permissions(권한) — Read: 보기·다운로드 / Write: 업로드·수정

FileZilla로 연결 확인. 루트 / = C:\inetpub\ftproot.

DNS 구축

  • 정방향(Forward) Zones = 도메인 → IP 레코드
  • 우클릭 → 존 추가 → Zone type: primary zone 선택
  • Zone 이름은 반드시 도메인 이름과 일치해야 작동
  • NS 레코드에 윈도우 서버 IP 지정

레코드: SOA(도메인 관리 설정), NS(담당 DNS 서버), A(IPv4 연결), AAAA(IPv6 연결), CNAME(다른 도메인으로 연결), www/file 등 서브도메인.

2. 리눅스 서버 — 패스워드 정책

dnf install -y libpwquality

비밀번호 복잡도 검사 기능. 최신 버전에는 대부분 기본 설치돼 있음.

vi /etc/security/pwquality.conf
difok = 1
minlen = 8
minclass = 0
maxrepeat = 0
  • difok — 이전 비밀번호와 최소 몇 자 이상 달라야 하는지
  • minlen — 비밀번호 최소 길이
  • minclass — 숫자/대문자/소문자/특수문자 중 최소 몇 종류 포함해야 하는지
  • maxrepeat — 같은 문자 연속 사용 제한
  • dcredit / ucredit / lcredit / ocredit — 각 문자 종류별 최소 포함 개수(음수면 강제, 양수면 보너스 개념)
