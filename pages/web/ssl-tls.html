SSL / TLS 정리

beebox 문제를 풀면서 TLS 개념이 계속 등장했는데 정확히 정리가 안 되어 헷갈려서 따로 정리한 노트

1. SSL/TLS 정의

네트워크에서 애플리케이션 계층 데이터(HTTP 등)를 안전하게 보호하는 보안 프로토콜.

  • SSL(Secure Sockets Layer) — 구버전 프로토콜
  • TLS(Transport Layer Security) — SSL을 개선한 최신 표준

현재는 TLS가 표준이지만, 관습적으로 SSL이라는 용어도 여전히 함께 쓰임.

2. TLS의 역할

  • 암호화(Encryption) — 데이터 내용 보호
  • 무결성(Integrity) — 데이터 변조 방지
  • 인증(Authentication) — 상대 서버/클라이언트 신원 확인

3. TLS가 보호하는 데이터

TLS는 "패킷 전체"가 아니라 HTTP 요청/응답 내용, 로그인 정보, 메시지·파일 데이터 등 애플리케이션이 실제로 주고받는 데이터(애플리케이션 계층 데이터)를 보호.

4. 동작 위치 (OSI 개념)

TLS는 특정 OSI 한 계층에 고정되지 않고, 애플리케이션 계층과 전송 계층(TCP) 사이에서 동작하는 논리적 보안 계층.

5. 동작 방식 (핵심 흐름)

  • 송신 측: 애플리케이션 데이터 → TLS가 암호화 → TCP로 전달
  • 수신 측: TCP 데이터 → TLS가 복호화 → 애플리케이션으로 전달

6. 헷갈렸던 핵심 정리

❌ "패킷 전체를 보호한다"
✔ 실제로는 애플리케이션 데이터만 보호. 네트워크 계층 정보(IP, TCP 등)는 통신을 위해 노출이 필요해 암호화하지 않음. 전체 통신을 보호하려면 VPN·IPsec 같은 기술이 추가로 필요.
❌ "OSI 6~7계층 사이에 물리적으로 존재한다"
✔ 실제로는 계층 사이에서 동작하는 논리적 구조일 뿐, 물리적으로 낀 계층이 아님.
❌ "애플리케이션 데이터와 전송 계층 데이터가 완전히 분리된다"
✔ 실제 구조는 계층별로 데이터가 추가되는 캡슐화(encapsulation) 구조.
TLS는 애플리케이션 계층 데이터를 암호화·인증·무결성 검증하여
전송 계층을 통해 안전하게 전달하는 보안 프로토콜
