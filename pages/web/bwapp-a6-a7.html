bwapp A6 · A7

민감한 데이터 노출(A6)과 기능별 접근 제어 미흡(A7) — OWASP Top 10 취약점 유형 정리

A6 — Sensitive Data Exposure (민감한 데이터 노출)

민감한 정보를 안전하게 저장·전송하지 않아 유출될 수 있는 취약점.

인코딩/평문 저장 취약점

Base64 Encoding(Secret) — Base64는 암호화가 아니라 단순 인코딩이라 누구나 쉽게 원본으로 복원 가능.
Text Files(Accounts) — 계정 정보를 암호화 없이 텍스트 파일에 저장, 와이어샤크로 캡처 시 평문 그대로 노출 확인.
HTML5 Web Storage(Secret) — LocalStorage/SessionStorage에 민감 정보 저장 시 XSS 발생하면 토큰·비밀번호 탈취 가능.

암호화 통신 미적용

Clear Text HTTP(Credentials) — 아이디/비밀번호를 HTTPS 대신 HTTP로 전송, 와이어샤크로 패킷 캡처해 평문 노출 확인. 스니핑 공격에 취약.

SSL/TLS 프로토콜 취약점

Heartbleed(CVE-2014-0160) — OpenSSL Heartbeat 버그로 서버 메모리(비밀번호, 세션쿠키, 개인키) 읽기 가능.
POODLE — SSL 3.0의 CBC 암호화 취약점으로 암호문 복호화, HTTPS 쿠키 탈취 가능.
SSL 2.0 Deprecated — 오래된 프로토콜로 여러 암호학적 약점 존재, TLS 사용 권장.
BEAST/CRIME/BREACH — 각각 TLS 1.0 CBC 모드, TLS 압축, HTTP 압축을 악용한 정보 탈취 공격.

Host Header Attack (Reset Poisoning)

HTTP Host 헤더를 조작해 비밀번호 재설정 링크를 공격자 서버로 유도. 요청 패킷을 가로채 Host 헤더값을 변경하면 사용자가 가짜 링크를 클릭했을 때 계정 탈취로 이어질 수 있음을 확인.

A7 — Missing Functional Level Access Control (기능별 접근 제어 미흡)

사용자 권한 검증이 부족해 허용되지 않은 기능·자원에 접근 가능한 취약점.

경로 조작 (Directory Traversal)

../를 이용해 의도된 범위를 벗어난 파일·디렉토리에 접근. 디렉토리 목록 조회형과 파일 직접 열람형(민감 파일 접근 성공) 두 가지로 실습.

파일 포함 (File Inclusion)

include() 등이 사용자 입력을 검증 없이 받아 의도하지 않은 파일을 코드로 실행. LFI=서버 내부 파일 포함, RFI=외부 서버 파일을 포함해 악성 코드 실행.

Host Header 공격 (Cache/Reset Poisoning)

Host 헤더를 조작해 악성 응답을 캐시에 저장시켜 다른 사용자에게 전달하거나, 비밀번호 재설정 링크를 공격자 서버로 유도.

접근 제어 취약점

Restrict Device Access — User-Agent 값으로 접근을 제한하려 하지만 값 자체가 조작 가능해 우회됨. 보안 목적이 아니라 UI 분기용으로만 써야 함.
Restrict Folder Access — 민감 페이지(admin 등)가 경로만 알면 인증 없이 접근 가능한 구조.

SSRF (Server Side Request Forgery)

서버가 사용자 입력값으로 대신 요청을 보내는 기능(미리보기 등)을 악용해, 외부에서는 접근 불가능했던 내부 자원에 서버를 매개로 접근.

XXE (XML External Entity)

XML의 외부 엔티티 기능을 이용해 서버 내부 파일을 가져오는 공격.
