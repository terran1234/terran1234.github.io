웹취약점 VM 풀이

워게임 사이트와 별도로 받은 가상머신(VM) 환경에서, 계정 취약점·쿠키 조작·CMS 설정 오류 등을 직접 찾아 공격한 실습 기록

1. 계정정보 취약 머신 (account01)

권한 상승 & 업로드 우회

회원가입 시 userLv 값을 직접 admin으로 변경해 관리자 권한 획득. .php 확장자는 업로드가 막혀있지만 .php3은 필터를 통과하고 실제로 실행되는 것을 확인해 웹쉘 업로드.

2. 계정정보 취약 머신 (account02)

정보 수정 취약점으로 관리자 탈취

회원가입 후 정보 수정 화면에서 이름·이메일·id만으로 본인 확인을 하고 비밀번호를 변경할 수 있는 구조를 발견 → 이 취약점으로 admin 비밀번호를 직접 변경해 로그인.

업로드 필터 우회 (확장자 트릭)

Apache는 파일명을 점(.)으로 쪼개 오른쪽부터 검사하며, mime.types와 httpd.conf의 AddType 목록 중 하나에 등록된 확장자를 처음 만나면 그걸로 처리 방식을 확정한다는 점을 이용.

파일명필터 통과실행이유
cmd.php❌-정확히 .php로 끝나 필터에 걸림
cmd.php.jpg✅❌맨 오른쪽 jpg가 먼저 판정됨
cmd.php.kr✅✅등록 안 된 확장자는 무시, 앞의 php가 적용됨
<?php system($_GET['cmd']); ?>

cmd.php.kr 형태로 업로드 후 ?cmd=id 파라미터로 명령 실행 성공 확인.

3. cookie 계열 머신

cookie-01

업로드 파일이 원본 이름 대신 타임스탬프+난수 조합으로 저장되는 구조. 쿠키 값에서 URL 인코딩된 경로(%2F = /)를 디코딩해 /imginfo/파일명0.txt 형태의 정보 파일 경로를 발견, 접근에 성공. 이미지 업로드 시 패킷을 가로채 내용을 수정한 뒤 전송하는 방식도 함께 실습.

4. wordpress 취약 머신

진행 중

취약하게 설치된 WordPress 환경을 재현한 실습용 머신. /etc/hosts 파일을 수정해 가상 도메인으로 접속하는 단계까지 진행.
