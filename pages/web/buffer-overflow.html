Buffer Overflow

메모리 취약점의 기본 원리와 오버플로우 공격 실습 — 인증 우회, 명령어 변조, 포맷 스트링 취약점, CTF 머신 풀이까지

1. 정의

프로그램이 데이터를 저장하는 공간(버퍼)의 크기를 확인하지 않고 너무 많은 데이터를 넣어서, 주변 메모리까지 덮어쓰는 취약점. 원래 코드 대신 공격자가 원하는 코드를 실행시키는 것이 목적.

2. 보호기법

  • ASLR — 메모리 주소를 랜덤화해 공격자가 주소를 예측하지 못하게 함
  • Stack Canary — 버퍼와 RET(Return Address) 사이에 감시값을 넣어 오버플로우를 탐지
  • NX bit — 스택 같은 영역에서 코드 실행을 막아 쉘코드 실행을 차단
  • PIE — 실행파일 자체 주소를 랜덤화해 함수·ROP 주소 예측을 어렵게 함
gcc -fno-stack-protector -z execstack -no-pie -o [파일명] [파일명].c

실습을 위해 위 옵션들로 보호기법을 의도적으로 비활성화 — 각각 Stack Canary 끄기, NX bit 우회(스택에서 실행 허용), PIE 끄기(주소 고정) 역할.

3. 기본 오버플로우 & Segmentation Fault

bof_vuln.c

char buffer[16];
strcpy(buffer, str);   // 길이 검사 없이 복사 — 취약점 발생 지점

16바이트보다 긴 문자열을 넣으면 Segmentation fault 발생. 패딩 크기에 따라 RET(복귀 주소)까지 도달하지 못하면 정상 출력 후 종료되고, 더 긴 입력으로 RET까지 덮어쓰면 그 시점에 비정상 종료됨을 확인.

4. 인증 우회 (Authentication Bypass)

bof_auth.c

volatile int auth_status = 0;
char password_buffer[16];
strcpy(password_buffer, argv[1]);   // 길이 제한 없음
if (strcmp(password_buffer, "secret123") == 0) auth_status = 1;

정답 비밀번호 없이도, 16바이트를 넘는 긴 입력으로 password_buffer를 넘쳐 바로 옆의 auth_status 변수 값까지 덮어써서 강제로 "인증 성공" 상태를 만드는 데 성공. 더 긴 입력에서는 이후 Segmentation Fault로 비정상 종료.

5. 시스템 명령어 변조

bof_system.c

char command[16] = "date";
char user_buffer[16];
strcpy(user_buffer, input);   // 취약점
system(command);

기본 실행 명령은 안전한 date였지만, user_buffer를 넘치게 입력해 이어진 메모리의 command 값을 whoami 등 다른 명령어로 덮어써서 의도치 않은 명령이 실행되는 것을 확인. 과도하게 침범하면 Segmentation Fault 발생.

6. 포맷 스트링 취약점

ASLR을 끄고(sysctl -w kernel.randomize_va_space=0) 실습. 포맷 지정자 %p(주소 Leak), %s(문자열 Leak), %n(메모리 값 Write)이 핵심.

fs_leak.c — 정보 유출

printf(user_input);   // 서식 없이 사용자 입력을 그대로 printf에 전달

%p.%p.%p... 형태로 입력하면 스택에 쌓인 메모리 주소 값들이 그대로 노출됨. %s를 섞으면 그 주소를 문자열로 읽으려다 잘못된 메모리 접근으로 Segmentation Fault가 나기도 함.

fs_write.c — %n으로 값 조작

원래 0(인증 실패)이어야 할 target_variable을 %n 포맷 지정자로 직접 덮어써서 값 자체를 조작하는 원리 실습(값 계산까지는 진행 중).

7. CTF — doubletrouble (VulnHub)

스테가노그래피부터 SQLi, 커널 취약점까지 여러 단계를 거치는 종합 실습.

1단계 — 초기 침투

gobuster로 /secret 페이지에서 이미지 파일 발견 → stegseek + rockyou.txt로 스테가노그래피 비밀번호 크래킹 성공, 숨겨진 계정 정보로 웹 로그인.

프로필 사진 업로드 기능에 php-reverse-shell.php 업로드 → 리스너로 리버스 쉘 연결 성공.

2단계 — 권한 상승 & 내부 VM 발견

sudo -l   // www-data가 비밀번호 없이 awk를 sudo로 실행 가능
sudo awk 'BEGIN {system("/bin/bash")}'   // root bash 획득

root 홈 디렉토리에서 doubletrouble.ova 파일을 발견해 반출, VirtualBox에 설치하니 "inner"라는 이름의 내부 전용 VM 확인.

3단계 — inner VM 침투

sqlmap -u http://[IP]/index.php -forms --dbs
sqlmap -u ... -D doubletrouble --tables
sqlmap -u ... -D doubletrouble -T users --dump

sqlmap으로 로그인 폼의 Time-Based Blind SQL Injection을 자동 탐지해 DB → 테이블 → 계정 정보까지 순서대로 덤프, 획득한 계정으로 SSH 접속해 user flag 획득.

4단계 — 커널 취약점으로 root 획득

uname -a로 커널 버전 확인 후 Dirty COW(Linux Kernel 2.6.22~3.9 PTRACE_POKEDATA Race Condition) 취약점 exploit 코드를 kali에서 받아 파이썬 서버로 전송.

gcc -pthread 40839.c -o dirty -lcrypt
./dirty
mv /tmp/passwd.txt /etc/passwd
su firefart

/etc/passwd에 UID 0(root 권한)의 firefart 계정을 새로 추가해 root 권한 획득, root flag까지 최종 확보.
