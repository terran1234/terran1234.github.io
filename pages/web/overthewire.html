overthewire — Natas

OverTheWire의 웹 해킹 워게임 Natas 시리즈 풀이 기록. URL의 문제번호만 바꾸면 다음 레벨로 접근 가능.

Level 0 → 1

페이지에 비밀번호가 있다는 언급이 있어 소스코드를 확인 → 페이지 소스코드에서 패스워드 직접 노출된 것을 발견.

Level 1 → 2

우클릭이 막혀있어 소스 보기를 막으려는 의도로 추정. 대신 개발자 도구(F12)로 최종 렌더링된 코드를 확인해 패스워드 획득.

우클릭=서버가 처음 보낸 원본 코드, 개발자 도구=자바스크립트 실행 후 최종 코드라는 차이를 이용.

Level 2 → 3

페이지엔 아무것도 없다고 안내되지만, 소스코드에 숨겨진 이미지 경로 발견 → 해당 경로(/files/users.txt)로 이동해 패스워드 획득.

Level 3 → 4

"구글도 못 찾을 것"이라는 힌트 주석 발견 → robots.txt 확인 → /s3cr3t/ 경로에서 users.txt 발견, 패스워드 획득.

Level 4 → 5

프록시로 진입 패킷을 확인해 Referer 헤더 값을 확인. Referer는 요청 직전 페이지 주소를 서버에 알려주는 값인데, 이걸 natas5가 보낸 것처럼 위조해서 통과.

Level 5 → 6

쿠키 loggedin 값을 0에서 1로 변경해 인증 우회.

Level 6 → 7

include "includes/secret.inc";
if($secret == $_POST['secret']) { ... }

secret 값을 별도 파일에서 불러와 비교하는 구조 확인 → includes/secret.inc 경로에 직접 접근해 소스코드로 secret 값 확인.

Level 7 → 8

?page 파라미터가 파일 경로로 그대로 쓰이는 것을 확인 (Path Traversal). 소스 주석에 있던 힌트 경로를 참고해 ../../../../../etc/natas_webpass/natas8를 파라미터에 삽입해 패스워드 파일 직접 열람.

Level 8 → 9

bin2hex(strrev(base64_encode($secret))) — Base64 인코딩 → 뒤집기 → 16진수 변환의 3단계 처리 확인. CyberChef(GCHQ 제작 데이터 변환 툴)에서 From Hex → Reverse → From Base64 순서로 Recipe를 구성해 역순으로 복원, secret 값 획득.

Level 9 → 10 — Command Injection

passthru("grep -i $key dictionary.txt");

needle 파라미터 값이 검증 없이 셸 명령어에 그대로 삽입되는 것을 확인. ;로 명령어를 분리해 추가 명령을 실행시키는 Command Injection 적용.

; cat /etc/natas_webpass/natas10

Level 10 → 11

if(preg_match('/[;|&]/', $key)) { ... }

이전 레벨의 Command Injection 문자(; | &)를 정규식으로 걸러내는 필터가 추가된 것을 확인. (우회 방법 분석 진행 중)

📌 Level 11까지 진행, 그 이후 레벨은 계속 풀이 중입니다.
