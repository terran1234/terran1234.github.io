beebox

bWAPP(Buggy Web Application)을 기반으로 한 취약한 웹 애플리케이션 실습 VM. Low/Medium/High 난이도별로 필터 우회 방식을 비교하며 실습

환경

Ubuntu 8.04 기반, Apache2 + MySQL + PHP 스택. OWASP Top 10을 포함한 수백 가지 취약점이 의도적으로 내장돼 있음. 초기 계정 bee/bug.

HTML Injection

Reflected (GET/POST/URL)

Low는 검증 전혀 없이 바로 삽입 성공. Medium은 < > 특수문자를 걸러내지만 URL 인코딩(%3C 등)으로 우회 — 필터는 인코딩된 값을 통과시키고, 브라우저가 디코딩하는 시점에 실제 태그로 해석되는 원리.

Stored (Blog)

입력값이 DB에 저장되어 이후 방문자 전체에게 실행되는 유형.

iFrame Injection

Low

ParamUrl 파라미터에 넣은 URL을 검증 없이 iframe으로 그대로 불러오는 구조. 임의 주소를 넣어 원치 않는 페이지를 삽입 가능.

OS Command Injection / PHP Code Injection

OS Command Injection

입력값이 명령어에 그대로 이어져 실행되는 것을 응답 결과로 확인 (Blind 유형도 존재).

PHP Code Injection

message 파라미터 값이 검증 없이 그대로 실행/출력되는 구조.

Server-Side Includes (SSI) Injection

서버가 SSI 명령어를 실행하고 결과만 브라우저에 전달하는 구조(클라이언트 사이드 스크립트와 반대). <!--#exec cmd="cat /etc/passwd" --> 형태로 서버 명령 실행 성공.

인증/세션 관리 취약점

CAPTCHA Bypassing

CAPTCHA를 한 번만 풀고 그 값을 재사용해서 우회.

Session Mgmt. — Administrative Portals

파라미터 값 변조(?admin=1)나 개발자 도구에서 쿠키의 admin 값을 직접 수정해 관리자 페이지 접근 성공.

Cookies (HTTPOnly / Secure) / Session ID in URL

HTTPOnly 미설정 시 XSS로 쿠키 탈취 가능, Secure 미설정 시 HTTPS가 아닌 채널로도 쿠키 전송, 세션 ID가 URL에 그대로 노출되는 문제들을 각각 확인.

📌 LDAP Connection, Mail Header Injection 등 일부 항목은 계속 진행하며 정리 예정입니다.
